# Internationalisation — Implémentation Technique

## Principe

Folia est une application mondiale — l'internationalisation doit être pensée dès le départ.
Chaque texte affiché à l'utilisateur passe par le système i18n — jamais de chaîne en dur dans le code.

---

## Stack

| Contexte | Librairie |
|---|---|
| Web (Astro) | `i18next` + `i18next-http-backend` |
| Back-office (React) | `react-i18next` |
| Mobile (React Native) | `react-i18next` + `i18next` |
| Back-end (Fastify) | `i18next` (messages d'erreur, emails) |

---

## Langues

### Langues de référence
- **Français** — langue principale du développement
- **Anglais** — langue de référence internationale

### Langues prioritaires au lancement
1. Français (`fr`)
2. Anglais (`en`)
3. Espagnol (`es`)
4. Allemand (`de`)
5. Portugais (`pt`)

### Ajout d'une nouvelle langue
Une langue n'est activée dans l'app que si elle atteint **90% de couverture** des clés critiques.
Les clés non traduites affichent la version anglaise par défaut.

## Structure des fichiers de traduction

```
apps/front-public/
  public/
    locales/
      fr/
        common.json        → textes génériques (boutons, navigation...)
        plants.json        → textes liés aux plantes
        errors.json        → messages d'erreur
        gamification.json  → grades, badges, points
      en/
        common.json
        plants.json
        errors.json
        gamification.json
```

### Format des clés

```json
// fr/plants.json
{
  "detail": {
    "title": "{{scientificName}}",
    "care": {
      "watering": "Arrosage",
      "wateringFrequency": "Tous les {{days}} jours",
      "light": "Lumière"
    },
    "toxicity": {
      "warning": "⚠️ Toxique pour {{animals}}",
      "safe": "Sans danger pour les animaux"
    }
  },
  "list": {
    "empty": "Aucune plante trouvée",
    "count_one": "{{count}} plante",
    "count_other": "{{count}} plantes"
  }
}
```

### Conventions de nommage des clés
- **camelCase** pour les segments
- Hiérarchie logique par feature
- Pluriel avec suffixe `_one`, `_other`, `_zero` (format ICU)
- Interpolation avec `{{variable}}`

## Utilisation dans le code

### React / React Native
```typescript
import { useTranslation } from 'react-i18next'

function PlantCard({ plant }: { plant: Plant }) {
  const { t } = useTranslation('plants')

  return (
    <View>
      <Text>{t('detail.care.wateringFrequency', { days: plant.care.wateringFrequencyDays })}</Text>
      {plant.isToxic && <Text>{t('detail.toxicity.warning', { animals: 'chats, chiens' })}</Text>}
    </View>
  )
}
```

### Pluriel
```typescript
t('list.count', { count: plants.length })
// count = 1 → "1 plante"
// count = 5 → "5 plantes"
```

### Dates et nombres
```typescript
// Toujours utiliser l'API Intl native — pas de librairie tierce
const formatter = new Intl.DateTimeFormat(locale, { dateStyle: 'long' })
formatter.format(new Date()) // "14 avril 2025" en fr, "April 14, 2025" en en

const numberFormatter = new Intl.NumberFormat(locale)
numberFormatter.format(1234.5) // "1 234,5" en fr, "1,234.5" en en
```

## Détection de la langue

### Ordre de priorité
1. Préférence sauvegardée dans le profil utilisateur
2. Paramètre URL (`?lang=fr`)
3. Header `Accept-Language` (web)
4. Langue système de l'appareil (mobile)
5. Français par défaut

---

## Chargement lazy

Les fichiers de traduction sont chargés à la demande par namespace :

```typescript
// Charger uniquement le namespace 'plants' sur la page plantes
i18n.loadNamespaces('plants')
```

---

## Gestion des traductions manquantes

```typescript
i18next.init({
  // Fallback vers l'anglais si la clé n'existe pas dans la langue courante
  fallbackLng: 'en',

  // Logger les clés manquantes en développement
  saveMissing: process.env.NODE_ENV === 'development',
  missingKeyHandler: (lng, ns, key) => {
    console.warn(`Missing translation: ${lng}/${ns}/${key}`)
  }
})
```

## Contribution communautaire

Les traductions communautaires sont gérées via **Crowdin** (ou équivalent) :
- Interface web pour les traducteurs non-développeurs
- Synchronisation automatique avec le repo via GitHub Action
- Validation par des locuteurs natifs avant merge
- Crédit des contributeurs visible dans l'app
